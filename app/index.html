Jon implements a hash table from his first week at Hack Reactor. Try it out!

Explanation

A hash table is a data structure that supports storage and retrieval of values with O(1) complexity, or constant time, meaning that in most cases the amount of time to store or retrieve a value does not increase as the data set grows. However, as the hash table becomes too full (over 75% utilization of indices) the hash table does need to double its size and reassign all values. This operation should be rather rare, and amortized over all other storage/retrieval transactions.
